学习记录 2018/06/29

​ hexo是一种本地搭建博客的工具，其博客的源文件都在自己的电脑里面，可以通过计算机端口去访问自己搭建的博客，过程只要不出现error或者Fatal就没什么问题，可能过程会出现 warn 但可以无需理会 ，直接走下一步就可以。

1、安装完git后，以管理员的方式在计算机开始的窗口运行git bash

2、通过git bash来安装Hexo，在git bash界面输入，输入完成后按回车等待。

等待一段时间后会出现一大段英文信息，其中只要不出现error、Fatal等信息就是没问题的。

3、将git bash 的操作界面转至桌面，输入完成后回车等待。

4、新建文件夹，然后将操作界面转至新建文件夹内。

5、此时操作的空间就在Blog文件夹内，同时可以看看桌面上有没有出现Blog这个文件夹，开始建立并初始化含hexo配置的文件夹myBlog。

第一个命令是初始化hexo博客框架的所有文件和所需的配置，第二个是进入myBlog文件，第三个是用git bash在myBlog内安装npm。

6、这些操作完成后，就能在Blog文件夹内看到myBlog的文件夹了，myBLog内的文件就是访问博客所需的一些配置文件。hexo充当的是一种自动打包生成博客文件的工具，其访问的源代码都在自己的电脑里，想要访问这个博客借助电脑端口即可。

说到这里，可能有的朋友会遇到一些小的问题，比如初始化myBlog时出现了错误，提示在myBlog文件夹内找不到node _modles这个模块，或者是在这个模块里找不到nodejs。

因为我在安装时也是碰到这个问题，查阅了很多资料都没解决，然后自己摸索着把它解决了。出现这个问题的第一种原因是安装时没有安装在默认位置，会出现这种情况，可以卸载后，安装在默认位置再试一遍。

第二种情况是默认位置安装还是会出现找不到那个模块，解决方法是，找到C:\Users\liun\AppData\Roaming\npm（这个是我电脑的路径，liun为电脑用户名。）打开这个文件夹，找到node_modules 把他复制到C:\Program Files\Git（这个路径是默认安装的路径，如果更改了路径就放到，安装时的那个路径内）文件夹内，再一次的运行命令就可以解决了。

7、通过git bash输入命令来获取访问端口

其中通过接口访问博客的原理是，源代码文件存放于自己的电脑里面，只需通过本机的端口就能访问到博客，而地址中的4000就是本机访问博客的端口号，当然了有些人的电脑可能会被一些软件占用了这个端口导致无法访问，这时可以通过git bash来修改端口号。

通过输入这个按回车确认等待一段时间，就把端口号改成5000了。然后按ctrl+c停止 就可以输入新的命令了。
